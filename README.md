# Vibe Coding

- There are multiple perceptions of what "vibe coding" means.
- Some say, it's about coding in a relaxed and enjoyable manner, focusing on the flow and creativity rather than strict deadlines or rigid structures.
- Others interpret it as coding that incorporates emotional intelligence, where the code reflects the mood or vibe of the user or the environment.

### Understanding Vibe Coding

- Vibe coding is a concept that incorporates the emotional intelligence of the developer into the coding process. It emphasizes creating software that resonates with users on an emotional level, enhancing user experience and engagement.

Explain in Simple terms:

- Vibe Coding means building software by describing what we want in natural language, and allowing AI to generate, modify or improve the code.

- In traditional coding, the developer writes most of the code manually.

- In Vibe Coding, the developer can describe the desired functionality or behavior in plain language, and AI tools can assist in generating the code, making it easier to implement complex features without needing to write every line of code from scratch.

Vibe coding = Developer idea + AI code generation + human review + testing + refinement.

#### Vibe Coding Workflow

1. Define the requirement
2. Give a structured prompt to the AI
3. Generate the first version of the code
4. Run and test the code
5. Identifiy issues
6. Refine the prompt or manually fix the code.

Example 1: Bad prompt

Make a portfolio website.

Problems:

- No tech stack
- No sections
- No design style
- No responsiveness requirement
- No content
- No output format.

Examplle 2: Good prompt

Act as a senior front-end developer.

Create a personal portfolio website using React and CSS.

Sections: 
1. Home
2. About Me
3. Skills
4. Projects
5. Contact

Requirements:
- Use functional components
- Use separate components for each section
- Use responsible layout
- Use clean and professional design
- Do not use Bootstrap or Tailwind
- Add sample data for projects
- Explain the folder structure

Output format:

1. Folder Structure
2. App.jsx
3. Component files
4. CSS files
5. Explanation of the code and design choices.
